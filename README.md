# Multithreaded HTTP Server

A modern C++ HTTP server project.

## Prerequisites

- **CMake** 3.23+
- **Git**
- **macOS**:
  - [Homebrew](https://brew.sh/)
  - Optional: LLVM (recommended for fuzzing): `brew install llvm`

- **Windows**:
  - Visual Studio 2022 (with C++ Desktop Development workload)

## Build

This project uses `vcpkg` as a submodule for dependency management.

1.  **Initialize the repository and submodule:**

    ```bash
    git submodule update --init --recursive
    # Bootstrap vcpkg (only needed once)
    # macOS/Linux:
    ./vcpkg/bootstrap-vcpkg.sh
    # Windows:
    # .\vcpkg\bootstrap-vcpkg.bat
    ```

2.  **Build using CMake Presets:**

    **Debug Build** (ASan+UBSan):
    ```bash
    cmake --preset debug
    cmake --build --preset debug
    ```

    **TSan Build** (best on Linux):
    ```bash
    cmake --preset tsan
    cmake --build --preset tsan
    ```

    **Fuzz Build** (libFuzzer + ASan):
    ```bash
    cmake --preset fuzz
    cmake --build --preset fuzz
    ```

    **Release Build** (Optimized):
    ```bash
    cmake --preset release
    cmake --build --preset release
    ```

## Run

```bash
./build/src/http_server
```

## Tests

```bash
ctest --preset debug
```

## Benchmarks

To run performance micro-benchmarks:

```bash
./build/benchmarks/server_benchmarks
```

## Fuzzing

The `fuzz` preset builds `fuzz/fuzz_target` when your compiler toolchain includes libFuzzer.

- On macOS, Apple Clang often does not ship the libFuzzer runtime. If the fuzz target is skipped,
  install Homebrew LLVM (`brew install llvm`) and configure with that `clang++`.

Run the fuzzer:

```bash
./build-fuzz/fuzz/fuzz_target -max_total_time=60
```

## Linting / formatting

- **Format**: `pre-commit run clang-format --all-files`
- **Tidy**: `pre-commit run clang-tidy --all-files` (requires `build/compile_commands.json`, generated by `cmake --preset debug`)

## clangd

This repo includes a `.clangd` that points clangd at `build/compile_commands.json` (generated by CMake presets).
